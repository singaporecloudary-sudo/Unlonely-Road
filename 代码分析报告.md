## 3. 图片加载机制

### 加载方式
- **方法**: new Image() （原生 JavaScript）
- **非 Canvas 绘制**: 使用 ctx.drawImage() 在 Canvas 上绘制

### 加载入口
- **函数**: AssetConfig.preloadAll()
- **位置**: asset-config.js 第 264-347 行
- **调用时机**: index.html 第 115 行，游戏启动时

### 图片存储
- **存储对象**: AssetConfig._loadedImages（JavaScript 对象字典）
- **键**: 完整路径字符串
- **值**: Image 对象

### 加载流程
1. **识别图片**: 遍历所有配置的路径
2. **创建 Image**: new Image()
3. **监听事件**: img.onload / img.onerror
4. **存储结果**: _loadedImages[path] = img
5. **回退渲染**: 图片不存在时使用占位符

### 主背景的加载路径
\\\
"assets/scenes/main_bg/" + "main_bg.png"
= "assets/scenes/main_bg/main_bg.png"
\\\

## 5. 如何修改让自定义图片生效

### 方案 A：替换图片文件（推荐）

1. **准备图片**
   - 格式: PNG、JPG 或 WebP
   - 尺寸: 建议 720x1280px（与画布相同）
   - 背景: 如需半透明效果，使用 PNG

2. **放置文件**
   `
   F:\游戏制作\合成飞车射击\assets\scenes\main_bg\main_bg.png
   `

3. **加载方式**
   - 游戏启动时自动调用 preloadAll()
   - 自动检测文件并使用 new Image() 加载
   - 加载成功则用 drawImage() 绘制真实图片
   - 加载失败则使用渐变占位符

4. **刷新生效**
   - 重新打开 index.html
   - preloadAll() 在页面加载时自动执行

### 方案 B：修改配置文件

1. **编辑文件**
   - 文件: F:\游戏制作\合成飞车射击\config\asset-config.js
   - 第 151-160 行的 mainBg 对象

2. **修改文件名**
   \\\javascript
   mainBg: {
     path: "assets/scenes/main_bg/",
     file: "your_custom_bg.png",  // 改为你的文件名
     size: { width: 720, height: 1280 },
     placeholder: { ... }
   }
   \\\

3. **保存配置**
   - 修改后保存 asset-config.js
   - 刷新页面，自动加载新配置

### 方案 C：使用占位符渐变（无需图片）

修改 placeholder 配置（已支持）：
\\\javascript
mainBg: {
  path: "assets/scenes/main_bg/",
  file: "main_bg.png",  // 不存在也不会报错
  placeholder: {
    type: "gradient",
    colors: ["#你的颜色1", "#你的颜色2", "#你的颜色3"],
    direction: "vertical"
  }
}
\\\
