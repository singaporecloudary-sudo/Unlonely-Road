CraftingScene.prototype._formatNumber = function(n) {
  if (!n) n = 0;
  n = Math.floor(n);
  if (n >= 1e9) return (n / 1e9).toFixed(1) + 'B';
  if (n >= 1e6) return (n / 1e6).toFixed(1) + 'M';
  if (n >= 1e3) return (n / 1e3).toFixed(1) + 'K';
  // 小于1000的整数直接返回，不要任何后缀
  return String(n);
};
---

  var cfg = this.ui.buttons.items[btnKey];

## 根本原因分析

观察 _formatNumber 函数的实现，问题症状为:
- UI显示 "118.6dd" 而不是预期的格式 (如 "118.6K"、"118.6M" 或 "118")

可能的原因:

### 假设 1: 客户端浏览器缓存陈旧的 crafting-scene.js 版本
- 旧版本的 _formatNumber 可能包含 'dd' 后缀代码
- 解决: Ctrl+F5 强制刷新浏览器缓存

### 假设 2: 服务器上存在旧的 crafting-scene.js 文件
- 服务器返回的是包含 'dd' 后缀的旧版本文件
- 解决: 确保服务器部署的是最新版本代码

### 假设 3: 缓存版本号 ?v=3 未被及时更新
- 当修改 crafting-scene.js 后，?v=3 版本号未变
- 用户的浏览器仍然加载旧的缓存版本
- 解决: 修改HTML中的版本号为 ?v=4 或更高

### 假设 4: 其他地方有全局的 formatNumber 方法覆盖了 _formatNumber
- 全局的 window.formatNumber 可能在加载后被修改
- 解决: 检查 localStorage、console错误、其他脚本注入

## 最终建议

1. 清空浏览器缓存并硬刷新 (Ctrl+Shift+Delete + Ctrl+F5)

2. 更新 index.html 中的版本号:
   从 ?v=3 改为 ?v=4:
   
   <script src="config/game-config.js?v=4"></script>
   <script src="config/asset-config.js?v=4"></script>
   <script src="src/core/engine.js?v=4"></script>
   <script src="src/core/state.js?v=4"></script>
   <script src="src/modules/crafting/crafting-scene.js?v=4"></script>
   <script src="src/modules/battle/battle-scene.js?v=4"></script>
   <script src="src/editor/component-editor.js?v=4"></script>

3. 在浏览器开发者工具 (F12) Network 标签中检查:
   - crafting-scene.js 是否从网络加载(200)或缓存(304)
   - 确保 Size 列显示的是最新文件大小

4. 在浏览器控制台执行以验证当前版本:
   console.log(CraftingScene.prototype._formatNumber.toString());
   
   应该显示:
   "function(n) {

   }"

5. 如果问题仍存在，检查是否有浏览器插件或用户脚本修改了 formatNumber
